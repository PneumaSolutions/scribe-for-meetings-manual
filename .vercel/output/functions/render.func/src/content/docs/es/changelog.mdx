Las versiones de RIM se denotarán mediante epígrafes en el nivel 2. Las actualizaciones sustanciales con varias funciones para ambas plataformas se dividirán en subtítulos Mac y Windows en el nivel 3.
## 3.2.13
### Windows
Se ha corregido un problema con la entrada de teclado al conectarse a máquinas de destino que ejecutan JAWS 2024.
## 3.2.12
Se ha solucionado un problema poco frecuente que podía impedir que la aplicación de RIM se conectara al servicio en la nube.

## 3.2.10

## 3.2.9
### macOS
Lamentablemente, ya no podemos ofrecer la entrada automática de micrófono estéreo en las conversaciones de voz en esta plataforma, ya que, en algunos casos, esto impedía que la entrada de micrófono funcionara en absoluto en un Mac. ## 3.2.8

## 3.2.6

## 3.1.43
### Mac
Esta actualización modifica la interfaz de usuario para las transferencias de archivos. Cuando se colocan uno o más archivos en el portapapeles a ambos lados de la conexión, el controlador recibe ahora una notificación indicando que la transferencia de archivos puede completarse a través del menú de RIM. Si el controlador elige esta opción de menú, se realiza la transferencia de archivos y, una vez finalizada, se abre en la máquina de destino una carpeta temporal que contiene el archivo o archivos transferidos. Este método de transferencia de archivos es necesario cuando la máquina de destino es un Mac. Cuando el destino es una máquina Windows, la transferencia de archivos puede completarse utilizando este nuevo método o el antiguo método de pegar en el Explorador de archivos de la máquina de destino.

## 0.12.0
Ahora puede crear un instalador preconfigurado que registrará automáticamente todas las máquinas en las que se instale como objetivos desatendidos en su cuenta de RIM.

Para ello, utilice el nuevo botón "Crear un instalador de destino preconfigurado", en la pantalla "Proporcionar ayuda". A continuación, deberá introducir un nombre base para la(s) máquina(s) de destino; RIM añadirá automáticamente el nombre de host de cada máquina de destino a este nombre base. Una vez registrados los equipos de destino, puede cambiarles el nombre como con cualquier equipo de destino desatendido. También puede establecer la fecha de caducidad del instalador, que debe estar comprendida entre 7 y 30 días. Una vez que RIM haya creado el instalador, obtendrás un enlace que puedes compartir directamente con los usuarios de destino, o descargarlo tú mismo para poder compartir el instalador de alguna otra forma.

Tenga en cuenta que cuando un usuario de destino ejecute el instalador preconfigurado, primero se le mostrará un cuadro de diálogo de confirmación que explica lo que hará el instalador. Este diálogo incluye el nombre del controlador que creó el instalador. Este paso de confirmación está diseñado para evitar abusos. Los clientes empresariales dispondrán de un modo completamente silencioso.

## 0.11.0
* La función Reiniciar y Reconectar ya está implementada. Utilizando la nueva opción Reiniciar y Reconectar del menú RIM del controlador, puedes realizar un reinicio estándar o de emergencia, y continuar automáticamente la sesión cuando el reinicio se haya completado. RIM también volverá a conectarse si la máquina de destino se reinicia por algún otro motivo, como cuando un instalador reinicia la máquina.
* RIM también intentará reconectarse automáticamente si la conexión se interrumpe por cualquier motivo. Esto ayudará con conexiones a Internet poco fiables.
* Cuando se ejecuta una sesión desatendida o se utiliza Reiniciar y reconectar, ahora se puede utilizar RIM mientras la máquina de destino está en la pantalla de inicio de sesión de Windows. Tenga en cuenta que RIM deberá volver a conectar la sesión cuando el equipo de destino pase de la pantalla de inicio de sesión al escritorio principal. Se trata de una limitación conocida de la implementación actual que es posible que solucionemos en una futura actualización.
* RIM es ahora totalmente funcional cuando el usuario de destino no tiene privilegios de administrador en el equipo de destino, siempre que el servicio Host de Remote Incident Manager se esté ejecutando en ese equipo. Tenga en cuenta que los privilegios de administrador siguen siendo necesarios para instalar las actualizaciones de RIM. Tenga en cuenta también que las nuevas versiones de RIM aún deben instalarse manualmente en equipos de destino desatendidos. Estamos trabajando en soluciones para estos dos problemas.
