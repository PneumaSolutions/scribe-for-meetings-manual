## 0.12.0
Agora é possível criar um instalador pré-configurado que registrará automaticamente todas as máquinas nas quais ele for instalado como alvos autônomos na sua conta do RIM.

Para fazer isso, use o novo botão chamado "Create a Pre-Configured Target Installer" (Criar um instalador de destino pré-configurado), na tela "Provide Help" (Fornecer ajuda). Em seguida, você precisará inserir um nome de base para a(s) máquina(s) de destino; a RIM adicionará automaticamente o nome do host de cada máquina de destino a esse nome de base. Depois que as máquinas de destino forem registradas, você poderá renomeá-las como qualquer outro destino não assistido. Você também pode definir a data de expiração do instalador, que deve estar entre 7 e 30 dias. Depois que a RIM criar o instalador, você receberá um link que poderá compartilhar diretamente com os usuários de destino ou fazer o download você mesmo para que possa compartilhar o instalador de outra forma.

Observe que, quando um usuário-alvo executar o instalador pré-configurado, ele mostrará primeiro uma caixa de diálogo de confirmação que explica o que o instalador fará. Essa caixa de diálogo inclui o nome do controlador que criou o instalador. Essa etapa de confirmação foi projetada para evitar abusos. Um modo totalmente silencioso estará disponível para clientes corporativos.

## 0.11.0
* O recurso Reboot and Reconnect (Reinicialização e reconexão) agora está implementado. Usando a nova opção Reboot and Reconnect (Reinicialização e reconexão) no menu RIM do controlador, você pode fazer uma reinicialização padrão ou de emergência e continuar automaticamente a sessão quando a reinicialização for concluída. O RIM também se reconectará se a máquina de destino for reinicializada por algum outro motivo, como quando um instalador reinicializa a máquina.
* O RIM também tenta se reconectar automaticamente se a conexão cair por qualquer motivo. Isso ajudará em conexões de Internet não confiáveis.
* Ao executar uma sessão autônoma ou usar Reboot and Reconnect, agora é possível usar o RIM enquanto o computador de destino estiver na tela de logon do Windows. Observe que o RIM precisará reconectar a sessão quando o computador de destino fizer a transição da tela de logon para a área de trabalho principal. Essa é uma limitação conhecida da implementação atual, que poderá ser resolvida em uma atualização futura.
* O RIM agora é totalmente funcional quando o usuário de destino não tem privilégios de administrador no computador de destino, desde que o serviço Remote Incident Manager Host esteja em execução nesse computador. Observe que os privilégios de administrador ainda são necessários para instalar as atualizações do RIM. Observe também que as novas versões do RIM ainda precisam ser instaladas manualmente em alvos não assistidos. Estamos trabalhando em soluções para esses dois problemas.
