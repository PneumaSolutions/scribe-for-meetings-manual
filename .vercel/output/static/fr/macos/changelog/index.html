Journal des modifications

Les versions de RIM sont désignées par des rubriques de niveau 2. Les mises à jour substantielles comportant plusieurs fonctionnalités pour les deux plates-formes seront divisées en sous-titres Mac et Windows au niveau 3.

3.2.13

Windows

Correction d’un problème de saisie au clavier lors de la connexion à des machines cibles exécutant JAWS 2024.

3.2.12

Résolution d’un problème rare qui pouvait empêcher l’application RIM de se connecter au service cloud.

3.2.11

Ajout d’un nouveau bouton dans la fenêtre Détails de la connexion pour demander que la machine cible envoie ses journaux RIM à Pneuma Solutions. N’utilisez cette fonction que si Pneuma Solutions vous le demande.

3.2.10

Correction d’un problème qui pouvait entraîner l’arrêt du fonctionnement de RIM jusqu’au prochain redémarrage de l’application si la connexion Internet de la machine était temporairement indisponible.

3.2.9

macOS

Nous ne sommes malheureusement plus en mesure de proposer l’entrée automatique du microphone stéréo dans les conversations vocales sur cette plateforme, car dans certains cas, cela empêchait l’entrée du microphone de fonctionner sur un Mac. Notez que ce changement n’a aucun effet sur la sortie audio capturée de la machine cible ; elle est toujours stéréo.

3.2.8

macOS

Correction d’un bogue qui empêchait RIM de s’afficher lors de l’activation de l’application via le Finder ou le Dock, au lieu d’utiliser le raccourci clavier global.

3.2.7

Les conversations vocales utilisent désormais automatiquement la stéréo lorsque le matériel audio le permet. Ceci est utile lorsque l’on utilise RIM pour la formation à la production audio.

3.2.6

Correction d’un problème récent qui pouvait empêcher RIM de fonctionner sur une nouvelle installation du système d’exploitation.

3.2.5

Correction du message affichant le temps libre restant.

3.2.4

La localisation de RIM dans d’autres langues est maintenant en version bêta. Les langues suivantes sont actuellement prises en charge : Allemand, Espagnol, Français, Italien et Suédois. Pour utiliser RIM dans l’une de ces langues, appuyez sur le bouton “Langue (BETA)” en bas de la fenêtre “Fournir de l’aide” ou “Recevoir de l’aide”, puis choisissez votre langue dans la liste déroulante et appuyez sur le bouton “Changer de langue”. Nous attendons avec impatience vos commentaires.

3.2.1

  • Le processus de démarrage d’une nouvelle session RIM du côté du contrôleur est désormais plus fluide. La fenêtre initiale, dans laquelle vous avez saisi votre mot-clé ou choisi une machine cible, ne disparaît pas tant que la fenêtre principale de la session n’est pas prête. Si la fenêtre initiale n’a plus le focus au démarrage de la session, par exemple si vous avez changé de tâche en attendant que l’utilisateur cible entre un mot-clé, RIM vous informera que la session est prête et la fenêtre principale de la session apparaîtra lorsque vous redonnerez le focus à RIM.
  • macOS uniquement : Dans le cadre du changement précédent, nous avons modifié la manière dont RIM s’ajoute et se supprime dans le Dock et la liste Commande+Tab. N’hésitez pas à nous faire savoir si vous rencontrez des problèmes avec le comportement actuel.
  • Cibles Windows uniquement : Si un lecteur d’écran est démarré sur la machine cible alors que l’accessibilité à distance est active, l’accessibilité à distance s’arrêtera discrètement afin d’éviter une double sortie vocale.

3.1.43

Mac

Cette mise à jour modifie l’interface utilisateur pour les transferts de fichiers. Lorsqu’un ou plusieurs fichiers sont placés dans le presse-papiers de part et d’autre de la connexion, le contrôleur reçoit désormais une notification indiquant que le transfert de fichiers peut être effectué via le menu RIM. Si le contrôleur choisit cette option de menu, le transfert de fichiers est effectué et, une fois terminé, un dossier temporaire contenant le ou les fichiers transférés est ouvert sur la machine de destination. Cette méthode de transfert de fichiers est nécessaire lorsque la machine de destination est un Mac. Lorsque la destination est une machine Windows, le transfert de fichiers peut être effectué en utilisant soit cette nouvelle méthode, soit l’ancienne méthode consistant à coller dans l’explorateur de fichiers sur la machine de destination.

    • nous n’avons pas encore déterminé comment, ou si, nous pourrons mettre en œuvre des installateurs de cibles préconfigurés, de sorte que vous devrez utiliser l’une des autres méthodes disponibles pour mettre en place un accès sans surveillance.

3.1.37

Cette mise à jour marque une étape importante dans le portage de RIM sur Mac. La fonction d’accessibilité la fonction d’accessibilité à distance de RIM est maintenant disponible lors de la connexion à des cibles Mac mac. En utilisant une configuration spéciale de VoiceOver sur la machine cible le contrôleur, qu’il utilise Windows ou Mac, entend la parole, alors que l’utilisateur cible n’entend rien tandis que l’utilisateur cible n’entend ni paroles ni effets sonores. Il n’y a pas non plus d’indication visible que VoiceOver il n’y a pas non plus d’indication visible que VoiceOver fonctionne sur la cible. Le processus de démarrage de l’accessibilité à distance, qu’il soit automatique ou manuellement, fonctionne de la même manière que pour les cibles Windows. Et bien sûr, à la fin de la session, la configuration spéciale de VoiceOver est conservée, lorsque la session se termine, la configuration spéciale de VoiceOver est arrêtée et VoiceOver est rétabli dans ses paramètres précédents.

Notez que les contrôleurs Windows et Mac doivent maintenant fonctionner avec la version 3.1.37 de version 3.1.37 pour bénéficier de toutes les fonctionnalités lors de la connexion à une cible Mac.

0.12.0

Vous pouvez maintenant créer un installateur préconfiguré qui enregistrera automatiquement toutes les machines sur lesquelles il est installé en tant que cibles sans surveillance dans votre compte RIM.

Pour ce faire, utilisez le nouveau bouton intitulé “Create a Pre-Configured Target Installer” (Créer un installateur de cibles préconfiguré), sur l’écran “Provide Help” (Fournir de l’aide). Vous devrez ensuite saisir un nom de base pour la ou les machines cibles ; RIM ajoutera automatiquement le nom d’hôte de chaque machine cible à ce nom de base. Une fois les machines cibles enregistrées, vous pouvez les renommer comme n’importe quelle machine cible non surveillée. Vous pouvez également définir la date d’expiration du programme d’installation, qui doit être comprise entre 7 et 30 jours. Une fois que RIM a créé le programme d’installation, vous recevez un lien que vous pouvez soit partager directement avec les utilisateurs cibles, soit télécharger vous-même afin de partager le programme d’installation d’une autre manière.

Notez que lorsqu’un utilisateur cible exécute le programme d’installation préconfiguré, il voit d’abord apparaître une boîte de dialogue de confirmation qui explique ce que le programme d’installation va faire. Cette boîte de dialogue contient le nom du contrôleur qui a créé le programme d’installation. Cette étape de confirmation est conçue pour éviter les abus. Un mode totalement silencieux sera disponible pour les entreprises.

  • RIM dispose maintenant d’une nouvelle option pour enregistrer une machine pour un accès sans surveillance sous votre compte RIM sans avoir à démarrer une session à distance à l’aide d’un mot-clé. Sur la machine que vous souhaitez enregistrer, appuyez sur le nouveau bouton appelé “Register This Machine for Unattended Access” (Enregistrer cette machine pour un accès sans surveillance), sur l’écran Receive Help (Recevoir de l’aide). Ensuite, connectez-vous à votre compte RIM si vous ne l’avez pas déjà fait sur cette machine, entrez un nom pour la machine, et vous avez terminé. Notez que nous travaillons toujours sur un moyen de personnaliser le programme d’installation de RIM, afin de permettre un déploiement de masse et de rendre aussi facile que possible la mise en place d’un accès sans surveillance sur des machines auxquelles vous n’avez pas encore accès.
  • RIM vous empêche désormais d’essayer d’ouvrir une connexion sans surveillance sur la même machine que celle sur laquelle vous exécutez le contrôleur.
  • Nous expérimentons actuellement un troisième raccourci clavier pour ouvrir le menu RIM : Windows+Shift+Backspace. Cela signifie qu’il y a maintenant trois options : Windows+Shift+Escape, Windows+Shift+Backspace, et Control+Shift+Backspace. Nous n’avons pas encore pris de décision définitive sur le ou les raccourcis clavier à conserver, mais nous le ferons avant la fin de la version bêta publique. Nous apprécions vos commentaires.
  • La plupart des options du menu RIM disposent désormais de raccourcis clavier à une lettre sur lesquels vous pouvez appuyer une fois le menu ouvert.
  • Cette mise à jour apporte une solution plus fiable aux problèmes intermittents signalés par les utilisateurs lorsque JAWS ne reconnaissait pas la saisie au clavier à partir de RIM. N’hésitez pas à nous faire savoir si vous continuez à avoir des problèmes avec ce scénario.

0.11.5

  • Cette mise à jour introduit deux nouveaux raccourcis clavier. Windows+Shift+D déconnecte la session lorsqu’elle est pressée d’un côté ou de l’autre. Pour la manette, Windows+Shift+F inverse la session.
  • Cette mise à jour ajoute des protections contre le blocage des modificateurs de clavier sur l’une ou l’autre machine.
  • Il existe une nouvelle option de menu, “Verrouiller la machine cible”, qui équivaut à appuyer sur Windows+L sur la cible.
  • RIM dispose désormais d’une fenêtre “À propos”, dans laquelle vous pouvez vérifier le numéro de version, contacter Pneuma Solutions pour obtenir de l’aide et des informations, lire le manuel et consulter le journal des modifications de la version.
  • RIM ne demande plus de commentaires à la fin d’une session ou lors de la fermeture de la fenêtre “Fournir de l’aide”.

0.11.4

Cette mise à jour corrige quelques bogues dans le processus de redémarrage et de reconnexion. Nous n’avons pas encore diagnostiqué tous les problèmes signalés, mais nous avons pu en trouver et en corriger quelques-uns lors de nos propres tests.

0.11.3

  • RIM se met désormais à jour en arrière-plan sur toutes les machines. RIM inclut désormais le numéro de la version installée dans l’infobulle de son icône dans la barre des tâches. Il s’agit de la dernière mise à jour que les utilisateurs devront installer manuellement, quelle que soit la configuration de la machine.
  • Si la machine cible est redémarrée, mais sans utiliser la fonction Reboot and Reconnect de RIM, RIM demande désormais au contrôleur s’il souhaite se reconnecter à la cible après le redémarrage. Si le contrôleur répond par la négative, la cible n’attendra pas une reconnexion après le redémarrage.

0.11.2

  • RIM se met désormais à jour en arrière-plan sur les machines cibles non surveillées. Il s’agit de la dernière mise à jour que vous devrez installer manuellement sur vos machines cibles sans surveillance.
  • Lors de la mise à jour avant une session interactive, RIM n’a plus besoin de passer par le contrôle de compte d’utilisateur. Cela signifie que les utilisateurs non administrateurs peuvent mettre à jour RIM une fois qu’il a été installé pour eux par un administrateur.

Il s’agit d’une mise à jour de correction de bogues. Veuillez noter que les cibles non surveillées doivent toujours être mises à jour manuellement en exécutant le dernier programme d’installation de RIM. Cependant, vous pourrez désormais vous connecter à une cible non surveillée une fois l’installation terminée, sans qu’il soit nécessaire de fermer la fenêtre “Recevoir de l’aide”.

0.11.0

  • La fonction de redémarrage et de reconnexion est désormais implémentée. En utilisant la nouvelle option Redémarrage et reconnexion du menu RIM du contrôleur, vous pouvez effectuer un redémarrage standard ou d’urgence, et poursuivre automatiquement la session une fois le redémarrage terminé. RIM se reconnecte également si la machine cible redémarre pour une autre raison, par exemple lorsqu’un programme d’installation redémarre la machine.
  • RIM tente également de se reconnecter automatiquement si la connexion est interrompue pour une raison quelconque. Cette fonction est utile dans le cas de connexions Internet peu fiables.
  • Lors de l’exécution d’une session sans surveillance ou de l’utilisation de Reboot and Reconnect, vous pouvez désormais utiliser RIM lorsque la machine cible se trouve sur l’écran d’ouverture de session Windows. Notez que RIM devra reconnecter la session lorsque la machine cible passe de l’écran de connexion au bureau principal. Il s’agit d’une limitation connue de l’implémentation actuelle que nous pourrons corriger dans une prochaine mise à jour.
  • RIM est désormais entièrement fonctionnel lorsque l’utilisateur cible ne dispose pas de privilèges d’administrateur sur la machine cible, pour autant que le service Remote Incident Manager Host soit exécuté sur cette machine. Notez que les privilèges d’administrateur sont toujours nécessaires pour installer les mises à jour de RIM. Notez également que les nouvelles versions de RIM doivent toujours être installées manuellement sur les cibles non surveillées. Nous travaillons sur des solutions à ces deux problèmes.
